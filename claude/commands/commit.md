# コミットコマンド

## 概要

ステージングされた変更（`git diff --cached`）を解析し、変更内容に応じた適切なprefix（feat:, fix:, chore: など）を付けた日本語のコミットメッセージを自動生成してコミットする。

## 引数フォーマット

```
$ARGUMENTS = [--prefix プレフィックス] [--message メッセージ] [--no-confirm] [その他の指示]
```

### オプション引数

- `--prefix <type>`: コミットprefix（feat, fix, chore など）を明示指定
- `--message <msg>`: コミットメッセージを明示指定（prefixは自動付与）
- `--no-confirm`: 確認なしでコミット実行
- その他: ユーザーが追加で指定した指示

## Prefixの種類

| Prefix | 用途 | 例 |
| :--- | :--- | :--- |
| `feat` | 新機能の追加 | 新しいコンポーネント、API、機能 |
| `fix` | バグ修正 | 不具合の修正、エラー対応 |
| `docs` | ドキュメントのみの変更 | README、コメント、JSDoc |
| `style` | コードの意味に影響しない変更 | フォーマット、空白、セミコロン |
| `refactor` | バグ修正でも機能追加でもないコード変更 | リファクタリング、構造改善 |
| `test` | テストの追加・修正 | テストコード |
| `chore` | ビルドプロセスやツールの変更 | 設定ファイル、依存関係 |
| `perf` | パフォーマンス改善 | 最適化 |
| `ci` | CI設定の変更 | GitHub Actions、CircleCI |

## 実行手順

### Step 1: 引数の解析

`$ARGUMENTS` から以下を抽出:

- `--prefix` オプション（指定があればそのprefixを使用）
- `--message` オプション（指定があればそのメッセージをベースに）
- `--no-confirm` フラグ（指定があれば確認をスキップ）
- その他のオプション・指示

### Step 2: ステージング状態の確認

#### 2-1: ステージングされた変更の確認

```bash
git diff --cached --name-only
```

#### 2-2: ステージングがない場合

```bash
git status --short
```

- **未ステージの変更がある場合**: 「ステージングされた変更がありません。`git add` で変更をステージングしてください」と案内
- **変更自体がない場合**: 「コミットする変更がありません」と表示して終了

### Step 3: 変更内容の解析

#### 3-1: diffの取得

```bash
# ステージングされた変更の詳細
git diff --cached --stat

# 変更内容の詳細（必要に応じて）
git diff --cached
```

#### 3-2: 変更の種類を判定

以下の優先順で判定:

1. **--prefix が指定されている場合**: そのprefixを使用
2. **新規ファイルが追加されている場合**:
   - 機能ファイル → `feat`
   - テストファイル → `test`
   - ドキュメントファイル → `docs`
   - 設定ファイル → `chore`
3. **既存ファイルの修正**:
   - バグ修正のパターン（fix, bug, error などの文脈） → `fix`
   - テストファイルのみ → `test`
   - ドキュメントファイルのみ → `docs`
   - 設定ファイルのみ → `chore`
   - その他の修正 → `refactor` または `feat`（変更内容による）

#### 3-3: コミットメッセージの生成

以下のフォーマットで生成:

```
<prefix>: <日本語の変更概要>
```

- 変更概要は50文字以内を目安
- 変更の「何を」したかを簡潔に記述
- 複数の変更がある場合は主要な変更を記述

### Step 4: ユーザーへの確認（--no-confirm でない場合）

生成したコミットメッセージを提示し、以下を確認:

1. prefixは適切か
2. メッセージ内容は正確か
3. 修正が必要な場合は修正案を聞く

**注意**: 確認は1回にまとめること。何度も聞かない。

### Step 5: コミットの実行

```bash
git commit -m "$(cat <<'EOF'
<prefix>: <コミットメッセージ>
EOF
)"
```

### Step 6: 結果の表示

- コミットが成功したことを通知
- コミットハッシュを表示
- 変更されたファイル数を表示

## 重要な注意事項

1. **ステージングのみ対象**: `git add` でステージングされた変更のみをコミット対象とする
2. **日本語**: コミットメッセージは日本語で記述
3. **prefix必須**: 必ずprefixを付ける
4. **確認フロー**: デフォルトでは確認してからコミット（`--no-confirm` で省略可能）
5. **ユーザー指示優先**: ユーザーが追加で指定した指示は最優先で適用

## 使用例

```bash
# 基本的な使用（変更を解析して自動生成）
/commit

# prefixを指定
/commit --prefix fix

# メッセージを指定（prefixは自動判定）
/commit --message ログイン機能を追加

# 確認なしでコミット
/commit --no-confirm

# 複合指定
/commit --prefix feat --message ユーザー認証機能を実装 --no-confirm
```
