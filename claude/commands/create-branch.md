# ブランチ作成コマンド（差分ベース）

## 概要

作業ツリー上の未コミット変更（ステージ済み + 未ステージ）を `git diff` で解析し、変更内容を表す適切なブランチ名を生成して新規ブランチを作成する。`git switch -c` で作成するため、現在の変更はそのまま新ブランチに持ち越される（コミットはしない）。

## 引数フォーマット

```
$ARGUMENTS = [補足情報テキスト(任意)]
```

- 追加の補足（背景・意図・チケット名など）があればブランチ名生成の参考にする。
- 引数なしでも動作する（diffのみで判断）。

## 実行手順

### Step 1: 差分の確認

以下のコマンドで現在の状態を取得する:

```bash
git status --short
git diff --stat
git diff --cached --stat
git diff
git diff --cached
```

判定:

- **差分が一切ない場合**: 「ブランチを作成する差分がありません」と表示して終了。
- **現在のブランチが master / main / develop 以外の作業ブランチの場合**: 既に作業ブランチ上にいる旨を警告し、AskUserQuestionで「新規ブランチを切るか / 現ブランチに留まるか」を1回だけ確認する。

### Step 2: ブランチ名の生成

diffの内容（変更ファイル・追加/削除された行・関数名など）と引数の補足から、以下のルールでブランチ名を生成する:

#### 命名ルール

- 差分の本質を表す簡潔な英語のケバブケース
- prefixは変更内容から推定する:
  - 新機能追加 → `feature/`
  - バグ修正 → `fix/`
  - リファクタ → `refactor/`
  - 設定・雑務 → `chore/`
  - ドキュメントのみ → `docs/`
  - テストのみ → `test/`
- 最大50文字以内
- 日本語のコメント・要件テキストは英訳して要約する

#### 例

| 差分内容 | ブランチ名 |
|---------|-----------|
| 新しい User Profile コンポーネントを追加 | `feature/add-user-profile` |
| login バリデーションのバグ修正 | `fix/login-validation` |
| README の更新のみ | `docs/update-readme` |
| settings.json の整理 | `chore/cleanup-settings` |
| 認証ロジックの抽出・関数化 | `refactor/extract-auth-logic` |

### Step 3: ユーザーへの確認

生成したブランチ名と、差分の要約・選定根拠をユーザーに提示し、AskUserQuestionで1回だけ確認する:

```
変更サマリ: {{diffから読み取った要約(1-2行)}}
変更ファイル: {{主要なファイル一覧}}
ブランチ名: feature/add-user-profile

このブランチ名で作成しますか？（変更があれば指定してください）
```

### Step 4: ブランチ作成

確認後、新規ブランチを作成する:

```bash
git switch -c {{ブランチ名}}
```

- 既存ブランチ名と重複する場合は `git switch -c` が失敗するため、エラーを表示して別名を提案する。
- 作業ツリーの未コミット変更はそのまま持ち越される。

### Step 5: 結果の表示

- 作成されたブランチ名
- 持ち越された差分のファイル一覧（`git status --short` の出力）
- 次の推奨アクションを案内（例: `git add -p` で選択的にステージング → `git commit`）

## 重要な注意事項

1. **コミットはしない**: ブランチの作成のみ。ステージング・コミットはユーザーに任せる。
2. **ブランチ名はユーザー確認必須**: 自動生成した名前を勝手に使わない。
3. **差分がない場合は実行しない**: 空のブランチを作成しない。
4. **既存ブランチの上書き禁止**: `git switch -c` が失敗したら、別名を生成して再確認する（`-C` での上書きは使わない）。
5. **質問は最小限**: Step 1の警告とStep 3の名前確認をまとめて1回で聞くのが望ましい。

## 使用例

```bash
# 差分のみで自動判断
/create-branch

# 補足情報を渡して精度を上げる
/create-branch ユーザープロフィールの表示機能

# チケットIDを補足として渡す
/create-branch HR-1234 ログインバリデーション修正
```
