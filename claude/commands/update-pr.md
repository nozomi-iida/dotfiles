# PR更新コマンド

## 概要

既存PRの本文を最新のgit diffとコミット履歴から再生成して更新する。
**ユーザーが手動で追記したFigmaリンク、Notionリンク、スクリーンショット、備考などは保持する。**

## 引数フォーマット

```
$ARGUMENTS = [--pr PR番号] [その他の指示]
```

### オプション引数

- `--pr <number>`: 更新対象のPR番号（省略時は現在のブランチのPRを自動検出）
- その他: ユーザーが追加で指定した指示

## 実行手順

### Step 1: 引数の解析

`$ARGUMENTS` から以下を抽出:

- `--pr` オプション（指定がなければ自動検出）
- その他のオプション・指示

### Step 2: 対象PRの特定

#### 2-1: PR番号の検出

`--pr` が指定されていない場合:

```bash
gh pr view --json number,title,body,baseRefName,headRefName
```

PRが見つからない場合はエラーを表示して終了。

#### 2-2: 既存PR本文の取得

```bash
gh pr view {{PR番号}} --json body,title,baseRefName --jq '.body'
```

既存PR本文を変数として保持する。

### Step 3: 既存PR本文から保護対象の抽出

以下のセクション・内容を既存PR本文から抽出し、保持する:

#### 3-1: リンク情報の保護

既存PR本文から以下を抽出して保持:

- **Figma URL**: `{{FIGMA_URL}}` プレースホルダーの前後にユーザーが追記したFigmaリンク
- **機能設計書 / Notion URL**: `{{FUNCTIONAL_SPECIFICATION}}` プレースホルダーの前後にユーザーが追記したNotionリンクやその他のリンク
- **チケット情報**: ユーザーが追記したチケットURL・IDなど

#### 3-2: スクリーンショット / 画面収録の保護

「スクリーンショット / 画面収録」セクションにユーザーが追記した画像・動画リンクをそのまま保持する。

#### 3-3: レビュー前チェックリストの保護

チェック済み状態（`[x]`）をそのまま保持する。

#### 3-4: 備考の保護

「備考」セクションにユーザーが追記した内容をそのまま保持する。

### Step 4: 最新diffの解析とPR本文の再生成

#### 4-1: ベースブランチの取得

PRのベースブランチを使用（Step 2-2で取得済み）。

#### 4-2: 差分の解析

```bash
# 変更ファイル一覧
git diff origin/{{base}}...HEAD --name-only

# 差分の詳細
git diff origin/{{base}}...HEAD --stat

# コミット履歴
git log origin/{{base}}..HEAD --oneline --no-merges
```

#### 4-3: diffの解析

`git diff origin/{{base}}...HEAD` の内容を読み、以下を更新:

- **この変更で実現すること**: 最新diffから再生成
- **実装方針・背景**: 最新diffから再生成
- **変更内容の詳細**: 最新のファイル一覧でテーブルを再生成
- **影響範囲**: 最新の変更ファイルから再推定

### Step 5: PR本文のマージ

再生成した内容と保護対象をマージして最終的なPR本文を組み立てる。

**更新するセクション:**

| セクション | 処理 |
| :--- | :--- |
| この変更で実現すること | 最新diffから再生成 |
| 実装方針・背景 | 最新diffから再生成 |
| 変更内容の詳細 | 最新diffから再生成 |
| 動作確認項目 | 最新diffから再生成 |
| 影響範囲 | 最新diffから再生成 |

**保護するセクション（既存PR本文をそのまま維持）:**

| セクション | 処理 |
| :--- | :--- |
| チケット情報（リンク含む） | 既存をそのまま維持 |
| Figma URL | 既存をそのまま維持 |
| 機能設計書 / Notion URL | 既存をそのまま維持 |
| スクリーンショット / 画面収録 | 既存をそのまま維持 |
| レビュー前チェックリスト | 既存をそのまま維持 |
| 備考 | 既存をそのまま維持 |

### Step 6: ユーザー確認

更新内容の概要を表示し、ユーザーに確認する:

- 変更されるセクションの差分サマリー
- 保護されるセクション一覧
- 「この内容でPR本文を更新しますか？」

### Step 7: プッシュとPR更新

#### 7-1: リモートへのプッシュ

ローカルがリモートより進んでいる場合:

```bash
git push
```

#### 7-2: PR本文の更新

```bash
gh pr edit {{PR番号}} --body "$(cat <<'EOF'
{{更新されたPR本文}}
EOF
)"
```

### Step 8: 結果の表示

- 更新されたPRのURLを表示
- 更新されたセクションと保護されたセクションの概要を通知

## 重要な注意事項

1. **保護対象の厳守**: Figmaリンク、Notionリンク、スクリーンショット、備考など、ユーザーが手動で追記した内容は絶対に消さない
2. **プレースホルダーの保護**: `{{TICKET}}`, `{{FIGMA_URL}}`, `{{FUNCTIONAL_SPECIFICATION}}` は絶対に変更しない
3. **セクション単位の更新**: 本文全体を置換するのではなく、セクション単位で更新/保護を判断する
4. **ユーザー確認必須**: 更新前に必ず変更内容を提示して確認を取る
5. **日本語**: PR本文は日本語で記述
6. **ユーザー指示優先**: ユーザーが追加で指定した指示は最優先で適用

## 使用例

```bash
# 現在のブランチのPRを自動検出して更新
/update-pr

# PR番号を指定して更新
/update-pr --pr 42

# 追加指示付き
/update-pr 実装方針セクションにキャッシュ戦略について追記して
```
