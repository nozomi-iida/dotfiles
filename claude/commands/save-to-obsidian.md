# Obsidian Tech メモ保存コマンド

## 概要

現在の会話から得られた技術的な学び・知見を整理し、ObsidianのTechフォルダにマークダウンファイルとして保存する。

## 引数フォーマット

```
$ARGUMENTS = [--title タイトル] [保存したい内容の補足説明]
```

### オプション引数

- `--title <title>`: ノートのタイトル（未指定の場合は会話内容から自動生成）
- その他: 保存したい内容に関する補足

## 実行手順

### Step 1: 会話の分析

現在の会話を振り返り、以下の観点で技術的な学びを抽出する:

- 新しく知った技術・ツールの使い方
- エラーの原因と解決方法
- 設定方法やコマンド
- ベストプラクティスや注意点

### Step 2: ユーザーへの確認

抽出した学びの要約を提示し、以下をまとめて確認する（質問は1回にまとめること）:

1. **保存内容**: 抽出した学びが正しいか、追加・修正したい点はあるか
2. **タイトル**: `--title` 未指定の場合、会話内容から生成した案を提示

### Step 3: ノートの生成

以下のフォーマットでマークダウンを生成する:

```markdown
## 概要
[学んだことの簡潔な説明]

## 詳細
[具体的な内容・手順・コード例など]

## 参考
[関連するドキュメントやリソースがあれば記載]
```

ルール:
- 会話の文脈がなくても理解できるよう、自己完結した内容にする
- コード例がある場合は必ず含める
- 不要な前置きや装飾は省く

### Step 4: ファイルの保存

#### 4-1: obsidian-vaultディレクトリの検索

ghq管理下にある `obsidian-vault` ディレクトリを探す:

```bash
find -L "$(ghq root)" -maxdepth 5 -type d -name "obsidian-vault" 2>/dev/null
```

見つからない場合はユーザーにパスを確認する。

#### 4-2: ファイルの書き込み

保存先: `{{見つかったobsidian-vaultのパス}}/Tech/{{タイトル}}.md`

- タイトルはファイル名としてそのまま使用する
- 同名ファイルが存在する場合はユーザーに確認する（上書き or タイトル変更）

### Step 5: 結果の表示

- 保存したファイルパスを表示
- 保存した内容の概要を表示

## 重要な注意事項

1. **会話の学びを抽出**: 単なる会話ログではなく、技術的な知見として再構成する
2. **自己完結**: 会話を読まなくても理解できるノートにする
3. **質問は最小限**: 1回にまとめて聞く
4. **日本語**: ノートは日本語で記述
5. **ユーザー指示優先**: ユーザーが内容を修正・補足した場合はそちらを優先

## 使用例

```bash
# 会話から自動で学びを抽出して保存
/save-to-obsidian

# タイトルを指定して保存
/save-to-obsidian --title "nvim-treesitterのv1.0移行対応"

# 補足説明付き
/save-to-obsidian Ctrl+bのキーマップ変更についても含めて
```
